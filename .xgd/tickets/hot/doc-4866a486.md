# Reference Capture Model

## 1. Purpose

Capture an external web page into a **structured, AI-facing reference bundle** that lets us: (1) point the system at an existing site and reproduce it as closely as the module catalog allows, (2) give the AI **eyes** on what it has created, and (3) build "a site like this one."

This document **supersedes** [[DOC-9]]'s reference-fetching model (static-first "Reference Digest", Layer A/B escalation, and the "one artifact, two audiences" convergence). Those are withdrawn. The Design Brief / site-memory concept in [[DOC-9]] is a separate, later concern and is unaffected.

## 2. Principles

1. **Rendered-only.** Most of the small-business web is JS-driven (Wix / Shopify / Squarespace). A headless browser that renders the page and is queried through its API is the *only* faithful way to see it. There is **no static HTML-extraction capture path.**
2. **Behind a Cloudflare-Browser-Rendering-shaped driver seam.** A pure `BrowserDriver` interface mirrors the CF Browser Rendering / `@cloudflare/puppeteer` surface; a **local Playwright** driver implements it now, a CF driver later. Migration = swap the driver; nothing above it changes.
3. **Navigate live → intercept-cache → query.** The browser navigates the *live* URL (so JS hydrates against its real origin), network interception caches **every** response (HTML, CSS, JS, images, fonts), then we query the browser for computed signals. **Never** `setContent()` a pre-fetched HTML shell — that re-creates the exact blindness of static capture.
4. **AI-facing, no human digest.** The capture is structured data for the AI. There is no dual-audience "readable digest" (that was a requirements misunderstanding).
5. **Exact where we can.** Copy is captured **verbatim**; colors, fonts, and assets are captured **exactly** (colors/fonts from *computed styles*, asset bytes mirrored). The AI's reproduction error is bounded to *structural mapping* only.
6. **Visibility-filtered.** Capture only what is actually rendered and visible; drop hidden chrome (cart drawers, region pickers, closed menus). Static structurally cannot do this.
7. **Style-scope segmentation.** A segment is a **styling context** — a region where one cluster of CSS applies (a "style signature": background + color scheme + type treatment + spacing/container). Boundaries fall where the signature shifts enough. Segments are **not** modules and **not** content groups. If the page is uniformly styled, **one segment is correct.**
8. **Catalog-decoupled.** The capture never mentions a module. The AI maps capture → modules at mapping time against the **live module registry**; sections that fit no module are logged as **gaps** that feed the module backlog. Add modules → nothing in the capture changes.
9. **Retained + archived.** The rendered HTML is kept as the AI's **escape hatch** (when the structured essence misses something, the AI checks the DOM). The whole bundle is archived, so capture can be **re-extracted / re-mapped offline** later without re-hitting the live site — "capture once, re-map forever."
10. **IP/copyright deferred.** `storage/references/` is gitignored (third-party material, uncommitted); the IP question is parked until the technical model is proven.

## 3. Pipeline

```
1c capture page <url>
  → driver.navigate(url)            live nav; JS hydrates
  → intercept + cache all responses HTML, CSS, JS, images, fonts
  → wait for network/idle
  → driver.query():                 computed styles, resolved background-image,
                                     document.fonts, bounding boxes, VISIBILITY
  → screenshot(full page)
  → segment by style signature      → style-scope sections
  → write capture bundle
```

Browser failure on a page → **retry**, never fall back to a blind static pipeline.

## 4. Capture bundle

```
storage/storage/references/<host>/<path>/            GITIGNORED (third-party)
  capture.json        structured essence — the AI's PRIMARY input
  screenshot.full.png the AI's eyes
  rendered.html       post-JS DOM (page.content()) — the AI's ESCAPE HATCH
  raw.html            original server response (kept; cheap)
  assets/             every mirrored subresource (images, fonts, css)
```

`capture.json` shape (catalog-agnostic; flat essence, never raw CSS):

- `theme` — EXACT tokens: `colors[]` (hex + usage + freq, from computed styles), `fonts[]` (family/role/weights + mirrored files), `typeScale`, `spacingScalePx`, `containerMaxWidthPx`.
- `sections[]` — each a **style-scope band**: `box`, `screenshot` (crop), `background` (`color | image | gradient` + optional `overlay {color,opacity}` — **text-over-image is first-class**), a flat `layout` (`textOverImage`, `contentAlign`, `arrangement`, `columns`, `contentMaxWidthPx`), `content[]` (role-tagged verbatim text runs, each carrying its exact color/font/size), and `items[]` (repeated sub-units like cards, flattened).
- `assets[]` — id, kind, original src, local path, dimensions, role.

## 5. Fidelity tiers

| Tier | Signals | Source | Reproduced |
|---|---|---|---|
| **Exact** | all copy, colors, fonts, asset bytes | computed styles + mirrored bytes | identically |
| **Measured** | spacing, container width, columns, section geometry | computed styles + bounding boxes | closely |
| **Segmented** | style-scope bands (input to AI mapping) | style-signature | AI maps → modules |

Segmentation errors are **grouping** errors, never content/fidelity errors: every text run and asset is captured with its own geometry regardless of boundaries, and the full-page screenshot is un-segmented ground truth — so a wrong boundary is advisory and re-segmentable, never destructive.

## 6. Screenshots — the AI's eyes

The same driver screenshots **our own** output to close the AI's perception loop: `1c render` a draft → `1c serve` it locally → screenshot the served page. Serving over localhost is what makes `/assets/` images load (fixing first-contact's blank-screenshot bug, which came from screenshotting a page that couldn't reach its own assets).

## 7. Mapping (future, AI)

Capture + the live module registry + the screenshot → a draft site (theme tokens set from exact values; sections mapped to modules) + a **gap log** of sections no module can represent. Gaps aggregated across many captures = a data-driven **module backlog**. The mapping is the AI's job and is out of scope for the capture tooling itself.

## 8. Migration to Cloudflare

Swap the Playwright driver for a CF Browser Rendering driver behind the same `BrowserDriver` seam; `storage/references/` bytes move to R2. The capture pipeline, schema, and bundle are unchanged.

## 9. Empirical basis

Validated against 5 real small-business homepages (Wix ×2, Shopify ×2, plain HTML ×1). On all 4 hosted-CMS sites the design-critical signals were **absent from static HTML**: colors behind `var()` indirection (one Wix site had 313 color-var refs), hero/background images as lazy 21px placeholders or not inline at all, and copy mixed with chrome. Only the single hand-coded site yielded usable static signals. This is the evidence base for rendered-only.
